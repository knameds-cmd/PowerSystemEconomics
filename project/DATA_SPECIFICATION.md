# 데이터 명세서 (Data Specification)

> 재생에너지 입찰제 도입에 따른 한국 육지계통 SMP 변화 분석 모델
> 더미 데이터 없이 실제 데이터로 실행하기 위한 전체 데이터 요구사항

---

## 개요

모델 실행에 필요한 데이터는 **8개 파일**로 구성되며, 모두 `data/raw/` 폴더에 배치합니다.

| # | 파일명 | 형식 | 필수 | 용도 |
|---|--------|------|------|------|
| 1 | `smp_demand.csv` | CSV | **필수** | SMP 실적 + 전력수요 시계열 |
| 2 | `재생에너지_발전량.csv` | CSV | **필수** | 태양광/풍력 발전량 시계열 |
| 3 | `generators.csv` | CSV | **필수** | 열발전 클러스터 정의 (9개) |
| 4 | `fuel_costs.csv` | CSV | **필수** | 연료별 단가 (월별) |
| 5 | `gencost.csv` | CSV | **필수** | 2차 비용함수 계수 (Piecewise Linear용) |
| 6 | `genthermal.csv` | CSV | **필수** | 기동비/최소가동시간 (Price Adder 검증용) |
| 7 | `nuclear_must_off.csv` | CSV | **필수** | 원전 계획정비 일정 |
| 8 | `marginal_fuel_counts.csv` | CSV | 선택 | 연료원별 SMP 결정횟수 (검증 비교용) |

## 파일 1: `smp_demand.csv` — SMP 실적 및 전력수요

### 용도
- Pre-ED 모델의 **보정 대상(actual SMP)**: Price Adder 보정 시 실제 SMP와 모형 SMP의 차이를 줄임
- **수요 프로파일**: 경제급전의 수급균형 우변 (수요 = 공급)

### 데이터 출처
- 한국전력거래소(KPX) 전력통계정보시스템 (EPSIS)
- 공공데이터포털: "계통한계가격(SMP)" / "전력수급현황"

### 형식

```
날짜,거래시간,smp_육지,수요_육지
2024-01-01,1,82150.5,48230.0
2024-01-01,2,79800.0,46150.0
...
2024-12-31,24,88000.0,51200.0
```

### 컬럼 상세

| 컬럼명 | 대체 가능 이름 | 자료형 | 단위 | 설명 |
|--------|---------------|--------|------|------|
| `날짜` | `date` | String / Date | `YYYY-MM-DD` | 거래일 |
| `거래시간` | `hour` | Int | 1~24 | **끝점 표시** (1 = 00:00~01:00 구간) |
| `smp_육지` | `smp_mainland`, `SMP`, `가격_육지` | Float64 | **원/MWh** | 육지계통 SMP |
| `수요_육지` | `demand_mainland`, `demand`, `수요` | Float64 | **MW** | 육지계통 전력수요 (시간당) |

### 주의사항
- 거래시간은 **1~24** (끝점 표시). 시간=1은 00:00~01:00 구간
- SMP 단위는 **원/MWh** (원/kWh가 아님에 주의)
- 수요 단위는 **MW** (MWh가 아님, 시간당 평균전력)
- 2024년 기준: 366일 × 24시간 = **8,784행**
- 결측치가 있으면 자동 선형보간 처리됨

### 데이터 범위 (2024년 한국 육지계통 참고)
- SMP: 약 30,000 ~ 250,000 원/MWh (평균 약 80,000~100,000)
- 수요: 약 32,000 ~ 80,000 MW (봄철 경부하 ~36,000, 여름 피크 ~78,000)

## 파일 2: `재생에너지_발전량.csv` — 태양광/풍력 발전량

### 용도
- Pre-ED: 음의 부하(negative load)로 처리 → 순수요 = 수요 - RE
- Post-ED: 입찰블록 생성의 기초 (태양광/풍력 **개별** 가용량)

### 데이터 출처
- 한국전력거래소(KPX) 전력통계정보시스템
- 공공데이터포털: "재생에너지 발전량"
- **이미 존재**: `재생에너지_발전량_2024.csv` (현재 프로젝트에 포함)

### 형식

```
날짜,거래시간,태양광_합계,풍력_육지
2024-01-01,1,0.0039,148.44
2024-01-01,2,0.0083,104.65
...
```

### 컬럼 상세

| 컬럼명 | 대체 가능 이름 | 자료형 | 단위 | 설명 |
|--------|---------------|--------|------|------|
| `날짜` | `date` | String / Date | `YYYY-MM-DD` | 거래일 |
| `거래시간` | `hour` | Int | 1~24 | 끝점 표시 |
| `태양광_합계` | `solar`, `solar_mainland` | Float64 | **MW** | 태양광 발전량 (전국 합산) |
| `풍력_육지` | `wind`, `wind_mainland` | Float64 | **MW** | 육지 풍력 발전량 |

### 주의사항
- **태양광과 풍력이 반드시 분리**되어야 함 (Post-ED에서 별도 블록 생성)
- 제주 풍력은 제외하고 **육지 풍력**만 사용
- 2024년 기준: 366일 × 24시간 = **8,784행**
- 야간 태양광은 0.0 (또는 극소량)

### 데이터 범위 (2024년 참고)
- 태양광: 0 ~ 약 30,000 MW (정오 피크)
- 풍력 (육지): 0 ~ 약 5,000 MW

## 파일 3: `generators.csv` — 열발전 클러스터 정의

### 용도
- 9개 열발전 클러스터의 물리적·경제적 파라미터 정의
- ED 모델의 공급측 구조 결정

### 데이터 출처
- 전력거래소 발전설비 현황 (EPSIS)
- 한국전력공사 통계
- 전력시장운영규칙 (비용평가 기준)
- 클러스터링: 동일 연료·유사 비용의 발전기를 합산하여 9개 그룹으로 구성

### 형식

```
name,fuel,pmin,pmax,ramp_up,ramp_down,heat_rate,vom,must_run,marginal_cost
Nuclear_base,nuclear,18000,24000,600,600,2.4,500,true,55000
Coal_lowcost,coal,6000,16000,2000,2000,2.1,2000,false,75000
Coal_highcost,coal,3000,10000,1500,1500,2.3,2500,false,90000
LNG_CC_low,lng,2000,12000,4000,4000,1.7,3000,false,110000
LNG_CC_mid,lng,1500,10000,3500,3500,1.8,3500,false,130000
CHP_mustrun,chp,4000,6000,1000,1000,2.0,2000,true,95000
LNG_GT_peak,lng,0,5000,5000,5000,2.5,5000,false,180000
Oil_peak,oil,0,2000,2000,2000,3.0,8000,false,250000
Hydro_fixed,hydro,0,4000,4000,4000,0.0,500,false,60000
```

### 컬럼 상세

| 컬럼명 | 자료형 | 단위 | 설명 |
|--------|--------|------|------|
| `name` | String | - | 클러스터 고유 이름 (다른 파일에서 매핑 키로 사용) |
| `fuel` | String | - | 연료원: `nuclear`, `coal`, `lng`, `oil`, `chp`, `hydro` 중 하나 |
| `pmin` | Float64 | **MW** | 클러스터 최소출력 (must-run이면 이 값 이상 강제) |
| `pmax` | Float64 | **MW** | 클러스터 최대출력 (해당 연료 전체 합산 설비용량) |
| `ramp_up` | Float64 | **MW/h** | 상향 램프 한계 (시간당 최대 출력 증가) |
| `ramp_down` | Float64 | **MW/h** | 하향 램프 한계 (시간당 최대 출력 감소) |
| `heat_rate` | Float64 | **Gcal/MWh** | 열소비율 (연료비 → 발전비 변환 계수) |
| `vom` | Float64 | **원/MWh** | 변동운영유지비 (정비비 + 기타변동비) |
| `must_run` | Bool | - | must-run 여부 (`true`/`false`). true면 항상 pmin 이상 발전 |
| `marginal_cost` | Float64 | **원/MWh** | 단순 한계비용 (Basic ED 폴백용: HR×연료비+VOM 근사) |

### 클러스터링 기준 및 산정 방법

**9개 클러스터 구성 권장안:**

| 클러스터 | 포함 발전기 | pmin 산정 | pmax 산정 |
|---------|-----------|----------|----------|
| Nuclear_base | 전체 원전 | Σ(호기별 Pmin) ≈ 75% × Pmax | Σ(호기별 정격용량) |
| Coal_lowcost | 저비용 석탄 (유연탄 하위 50%) | Σ 기술적 최소출력 | Σ 정격용량 |
| Coal_highcost | 고비용 석탄 (유연탄 상위 50%) | Σ 기술적 최소출력 | Σ 정격용량 |
| LNG_CC_low | LNG 복합 저비용 (하위 50%) | Σ 기술적 최소출력 | Σ 정격용량 |
| LNG_CC_mid | LNG 복합 고비용 (상위 50%) | Σ 기술적 최소출력 | Σ 정격용량 |
| CHP_mustrun | 집단에너지/열병합 (열 의무공급) | Σ 열 공급 최소출력 | Σ 정격용량 |
| LNG_GT_peak | LNG 가스터빈 (첨두용) | 0 (기동정지 자유) | Σ 정격용량 |
| Oil_peak | 유류 발전 | 0 | Σ 정격용량 |
| Hydro_fixed | 수력/양수 | 0 | Σ 가용 용량 |

**파라미터 산정:**
- `heat_rate`: 해당 클러스터 발전기의 **가중평균** 열소비율
- `vom`: 전력시장운영규칙의 변동운영비 기준 가중평균
- `ramp_up/down`: 클러스터 내 발전기의 **합산** 램프 능력
- `marginal_cost`: `heat_rate × 연료단가 + vom` (대략치, Basic ED 폴백용)

## 파일 4: `fuel_costs.csv` — 월별 연료 단가

### 용도
- 유효 한계비용 계산: `c̃ = HR × FuelPrice + VOM`
- 월별 변동 반영 (계절에 따른 연료비 변화)

### 데이터 출처
- 한국전력거래소: 월별 연료비 단가 공고
- 에너지경제연구원: 에너지통계월보
- 광물자원공사: 석탄/LNG 수입가격

### 형식

```
year_month,fuel,fuel_cost
2024-01,nuclear,5000
2024-01,coal,30500
2024-01,lng,54000
2024-01,oil,72000
2024-01,chp,46000
2024-01,hydro,0
2024-02,nuclear,5000
2024-02,coal,29800
...
```

### 컬럼 상세

| 컬럼명 | 자료형 | 단위 | 설명 |
|--------|--------|------|------|
| `year_month` | String | `YYYY-MM` | 해당 연월 |
| `fuel` | String | - | 연료원 이름 (`generators.csv`의 `fuel` 컬럼과 일치) |
| `fuel_cost` | Float64 | **원/Gcal** | 해당 월의 연료 단가 |

### 연료 단가 참고 범위 (2024년 기준)

| 연료 | 단위가격 범위 (원/Gcal) | 비고 |
|------|------------------------|------|
| `nuclear` | 4,500 ~ 5,500 | 핵연료, 안정적 |
| `coal` | 25,000 ~ 35,000 | 유연탄, 국제가격 연동 |
| `lng` | 45,000 ~ 65,000 | LNG, 변동성 큼 |
| `oil` | 60,000 ~ 85,000 | 유류, 국제유가 연동 |
| `chp` | 40,000 ~ 50,000 | LNG 기반 열병합 |
| `hydro` | 0 | 연료비 없음 (기회비용은 VOM에 포함) |

### 주의사항
- **월별 1행**: 12개월 × 6개 연료 = **72행** (연간)
- 연료명은 `generators.csv`의 `fuel` 컬럼과 **정확히 일치**해야 함
- 단위는 반드시 **원/Gcal** (원/MWh가 아님!)
- 월별 변동이 없으면 모든 월에 동일 값을 넣어도 무방

## 파일 5: `gencost.csv` — 2차 비용함수 계수

### 용도
- Piecewise Linear 비용함수 생성 (개선사항 2)
- `C(P) = a × P² + b × P + c` → S개 구간으로 선형 근사

### 데이터 출처
- MATPOWER `mpc.gencost` 데이터
- 한국전력거래소 비용평가 자료
- 학술논문의 발전기 비용함수 추정치

### 형식

```
name,a,b,c
Nuclear_base,0.0005,12000,50000
Coal_lowcost,0.002,63000,80000
Coal_highcost,0.003,72000,70000
LNG_CC_low,0.004601,50243,5213
LNG_CC_mid,0.0055,96500,6000
CHP_mustrun,0.0035,80000,40000
LNG_GT_peak,0.01,150000,3000
Oil_peak,0.008,210000,5000
Hydro_fixed,0.0,60000,0
```

### 컬럼 상세

| 컬럼명 | 자료형 | 단위 | 설명 |
|--------|--------|------|------|
| `name` | String | - | 클러스터 이름 (`generators.csv`의 `name`과 일치) |
| `a` | Float64 | **원/(MW²·h)** | 2차항 계수 |
| `b` | Float64 | **원/MWh** | 1차항 계수 (= P=0에서의 한계비용) |
| `c` | Float64 | **원/h** | 상수항 (무부하비에 해당) |

### 수학적 의미

```
총 비용: C(P) = a·P² + b·P + c  [원/h]
한계비용: MC(P) = dC/dP = 2a·P + b  [원/MWh]
```

- `a > 0`: 출력 증가 시 한계비용이 증가 (볼록 비용함수)
- `a = 0`: 선형 비용함수 (한계비용 일정 = b)
- `b`: P=0에서의 기저 한계비용
- `c`: 무부하비 (발전하지 않아도 발생하는 고정비)

### MATPOWER 형식 참조

MATPOWER `mpc.gencost` 행 구조:
```
[type, startup, shutdown, n, c_{n-1}, c_{n-2}, ..., c_0]
```
- Type=2: 다항식, n=3(2차)이면 계수 3개
- 예: `[2, 47398.56, 0, 3, 0.004601, 50243, 5213]`
  → `a=0.004601`, `b=50243`, `c=5213`

### 주의사항
- 클러스터 수만큼 **9행** (헤더 제외)
- `a`가 매우 작은 값 (0.001~0.01 수준)이 일반적
- 수력은 `a=0` (선형)
- `name`이 `generators.csv`와 일치해야 함

## 파일 6: `genthermal.csv` — 기동비 및 최소가동시간

### 용도
- Price Adder의 **물리적 합리성 검증** (개선사항 1)
- `adder_max = startup_cost / min_up_time / pmax_unit`
- 교차검증 시 adder clamp 범위로 사용

### 데이터 출처
- MATPOWER `mpc.genthermal` 데이터
- 전력거래소 발전기 기동정지 비용 자료
- 발전사업자 제출 비용자료

### 형식

```
name,startup_cost,min_up_time,pmax_unit
Nuclear_base,0,72,1000
Coal_lowcost,120000,8,500
Coal_highcost,100000,6,500
LNG_CC_low,47398.56,4,880
LNG_CC_mid,52138.42,4,700
CHP_mustrun,30000,6,200
LNG_GT_peak,10000,1,150
Oil_peak,15000,1,200
Hydro_fixed,0,1,500
```

### 컬럼 상세

| 컬럼명 | 자료형 | 단위 | 설명 |
|--------|--------|------|------|
| `name` | String | - | 클러스터 이름 (`generators.csv`의 `name`과 일치) |
| `startup_cost` | Float64 | **천원** | 고온기동비 (Hot Start Cost) |
| `min_up_time` | Float64 | **시간(h)** | 최소가동시간 (UT) |
| `pmax_unit` | Float64 | **MW** | 대표 호기의 정격용량 (클러스터 합산이 아님!) |

### 물리적 검증 공식

```
adder_max [원/MWh] = startup_cost × 1000 / min_up_time / pmax_unit
```

**검증 예시:**
- LNG_CC_low: `47,398.56 × 1000 / 4 / 880 = 13,466 원/MWh`
- Price Adder가 이 값의 1.5배 이내이면 물리적으로 합리적

### MATPOWER genthermal 형식 참조

```
[type, UT, DT, inistate, initialpower, ramp_up, ramp_down,
 startup_limit, shutdown_limit, startup1, startup2, startup3,
 startupdelay1, startupdelay2, startupdelay3]
```
- `startup1` = 고온기동비 (천원) → 이 값을 `startup_cost`로 사용
- `UT` = 최소가동시간 → `min_up_time`
- `startup2/3` = 온온/냉온 기동비 (참고용, 본 모델에서는 startup1만 사용)

### 주의사항
- `startup_cost` 단위는 **천원** (원이 아님!)
- `pmax_unit`은 **클러스터 합산 Pmax가 아닌 대표 호기 1기의 용량**
- 클러스터 수만큼 **9행**

## 파일 7: `nuclear_must_off.csv` — 원전 계획정비 일정

### 용도
- 분석 대상일의 **가용 원전 용량** 동적 조정 (개선사항 3)
- 정비 호기 수에 따라 Nuclear_base 클러스터의 Pmax/Pmin 변경

### 데이터 출처
- 한국수력원자력 계획예방정비 일정
- 전력거래소 발전정비 계획 공고
- 전력시장운영시스템 (전력거래소)

### 형식

```
id,off_start_day,off_start_time,off_end_day,off_end_time
1,45,1,95,24
2,60,1,110,24
3,1,1,341,24
5,120,1,160,24
8,200,1,250,24
```

### 컬럼 상세

| 컬럼명 | 자료형 | 단위 | 설명 |
|--------|--------|------|------|
| `id` | Int | - | 원전 호기 고유번호 (임의 번호 가능) |
| `off_start_day` | Int | **연중 일수 (1~365)** | 정비 시작일 (1=1월1일) |
| `off_start_time` | Int | **시 (1~24)** | 정비 시작 시각 |
| `off_end_day` | Int | **연중 일수 (1~365)** | 정비 종료일 |
| `off_end_time` | Int | **시 (1~24)** | 정비 종료 시각 |

### 가용 용량 계산 공식

```
특정 날짜(day)에 대해:
  offline_count = Σ (off_start_day ≤ day ≤ off_end_day인 호기 수)
  available_pmax = (total_units - offline_count) × unit_capacity
  available_pmin = available_pmax × 0.75
```

**모델 가정:**
- 총 원전 호기 수: **24기** (기본값, 코드에서 조정 가능)
- 호기당 정격용량: **1,000 MW** (기본값, 코드에서 조정 가능)
- 최소출력 비율: **75%** (한국 원전 운전 관행)

### 주의사항
- **1행 = 1개 호기의 1회 정비 기간** (같은 호기가 연중 2회 정비하면 2행)
- 연중 대부분 정비인 호기(예: 1일~341일)는 영구정지 또는 장기정비 호기
- 봄철(3~5월, 약 60~150일)에 정비 호기가 집중되는 것이 일반적
- 행 수는 가변 (정비 호기 수에 따라)
- **날짜 표현**: 연중 일수 (Day of Year). 1월1일=1, 12월31일=365

### 연중 일수 변환 참고

| 월 | 시작일 | 종료일 |

| 8월 | 213 | 243 |

## 분석 대상일 선정

모델은 **대표일 단위** (24시간)로 실행됩니다. 연간 데이터에서 계절별 3일씩 **12일**을 선정합니다.

```
선정 기준 (계절별):
1. 경부하·고재생일: 태양광+풍력 점유율 최대 & 수요 하위 50%
2. 최대부하일: 일별 최대수요가 가장 큰 날
3. 평균일: 일평균 SMP가 중앙값에 가장 가까운 날
```

**교차검증 시**: 12일 중 Leave-One-Out으로 11일 훈련 / 1일 테스트

## 단위 요약표

| 물리량 | 단위 | 주의 |
|--------|------|------|
| 전력 (수요, 발전량, Pmin/Pmax) | **MW** | MWh가 아님 |

| REC 가격 | **원/kWh** | → 원/MWh 변환 시 ×1000 |
| 램프율 | **MW/h** | 시간당 출력 변화 |
| 시간 (거래시간, UT/DT) | **시간(h)** | 1~24 |
| 날짜 | `YYYY-MM-DD` 또는 연중일수 | 파일별 상이 |
